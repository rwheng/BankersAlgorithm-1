Banker's Algorithm

Banker's Algorithm is an algorithm for avoiding deadlocks. Its' main purpose is to determine if system in a given state can run all processes.

How to:

On the left is the initial configuration of the system. This contains the information in the same format as detailed on the D2L page. On the right is the form to request new resources to be allocated to to a process. The first text box is the process ID (0-4 in the default config) and the second box is the resource request field (3 space separated integers in the default configuration). The bottom text area is a read only log of the processes that went on within the bankers algorithm to determine if the state is safe or not. If either new resources are requested or the safety of the system is queried a banner will pop up below this div that details the safe-ness of the current system.
